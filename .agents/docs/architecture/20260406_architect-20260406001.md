# 密码卡片操作按钮功能架构设计

## 架构概览

本需求为密码卡片添加编辑、复制、删除操作按钮，采用悬停显示设计。架构设计遵循 PassLock 现有的 Electron 主进程/渲染进程分离原则，保持安全架构约束。

### 整体架构图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           渲染进程 (Vue 3)                               │
├─────────────────────────────────────────────────────────────────────────┤
│  VaultPage.vue                                                          │
│  ├─ 状态管理                                                            │
│  │  ├─ editingEntry: VaultEntry | null  (当前编辑条目)                  │
│  │  ├─ showDeleteConfirm: boolean       (删除确认对话框)                │
│  │  └─ deletingEntryId: string | null   (待删除条目ID)                  │
│  ├─ 事件处理                                                            │
│  │  ├─ handleEditEntry(entryId)                                        │
│  │  ├─ handleDeleteEntry(entryId)                                      │
│  │  └─ handleConfirmDelete()                                           │
│  └─ 组件树                                                              │
│     ├─ PasswordCard (卡片组件，emit事件)                                │
│     ├─ AddPasswordModal (编辑弹窗，扩展编辑模式)                        │
│     └─ ConfirmDialog (删除确认对话框，新增)                             │
├─────────────────────────────────────────────────────────────────────────┤
│  VaultStore (Pinia)                                                     │
│  ├─ editEntry(id, input) → 解密+加密+更新                               │
│  ├─ deleteEntry(id) → 删除条目                                          │
│  └─ getDecryptedPassword(entryId) → 解密密码供编辑                      │
├─────────────────────────────────────────────────────────────────────────┤
│                          IPC 桥接层 (preload)                           │
├─────────────────────────────────────────────────────────────────────────┤
│                          主进程 (Electron)                              │
├─────────────────────────────────────────────────────────────────────────┤
│  crypto.ts                                                              │
│  ├─ encrypt(plaintext, masterKey)                                      │
│  └─ decrypt(encryptedData, masterKey)                                  │
│  database.ts                                                            │
│  ├─ updateEntry(id, updates)                                            │
│  └─ deleteEntry(id)                                                     │
└─────────────────────────────────────────────────────────────────────────┘
```

### 编辑流程

```
┌──────────┐    click edit     ┌──────────────┐
│PasswordCard│ ──────────────→ │ VaultPage    │
└──────────┐                   │ handleEdit   │
           │                   └──────────────┘
           │                          │
           │                          ├─→ VaultStore.getDecryptedPassword(entryId)
           │                          │       │
           │                          │       └─→ IPC: crypto:decrypt
           │                          │
           │                          ├─→ editingEntry = entry
           │                          ├─→ showEditModal = true
           │                          │
           │                          ↓
           │                   ┌──────────────┐
           │                   │AddPassword   │
           │                   │Modal (edit)  │
           │                   │预填充数据    │
           │                   └──────────────┘
           │                          │
           │                     click update
           │                          │
           │                          ├─→ VaultPage.handleUpdateEntry(input)
           │                          │       │
           │                          │       └─→ VaultStore.editEntry(id, input)
           │                          │               │
           │                          │               ├─→ IPC: crypto:encrypt (新密码)
           │                          │               │
           │                          │               └─→ IPC: db:updateEntry
           │                          │
           │                          ├─→ showEditModal = false
           │                          ├─→ editingEntry = null
           │                          │
           │                          ↓
           │                   ┌──────────────┐
           │                   │ PasswordCard │
           │                   │ 显示更新内容 │
           │                   └──────────────┘
```

## 安全架构

### 安全边界定义

| 边界 | 描述 | 信任级别 |
|------|------|----------|
| **主进程** | 加密/解密操作执行环境 | ✅ 可信任 |
| **渲染进程** | UI展示、临时密钥存储 | ✅ 可信任 |
| **IPC通道** | Electron内部进程通信 | ✅ 可信任 |
| **操作系统剪贴板** | 数据可能被其他应用读取 | ⚠️ 半信任 |
| **文件系统** | 数据可能被其他应用访问 | ⚠️ 半信任 |

### 密钥传递机制（修正声明）

**⚠️ 重要修正**：经 Cryptographer 审计，原架构声明与实际实现不一致。

**实际实现**：
- 渲染进程通过 IPC 调用时传递 masterKey 字符串
- 主进程接收 masterKey 执行加密/解密操作
- IPC 通道在 Electron 可信边界内（同一应用的不同进程）

**安全评估**：
- Electron IPC 在可信边界内，当前实现可接受
- 长期可考虑 Session 模式重构（主进程维护加密session，渲染进程发送请求ID）
- 本次任务不涉及 IPC 架构重构

### 加密参数修正

根据 Cryptographer 审计建议，加密模块需进行以下修正：

| 参数 | 当前值 | 修正值 | 原因 |
|------|--------|--------|------|
| IV_LENGTH | 16字节 | **12字节** | NIST SP 800-38D 推荐值，AES-GCM 最佳实践 |

**迁移策略**：
1. 解密时自动检测 IV 长度，兼容旧格式（16字节）和新格式（12字节）
2. 新加密数据使用 12 字节 IV
3. 编辑条目时自动使用新格式重新加密

### 编辑模式安全设计

**安全约束**：
- 解密后的密码仅在编辑弹窗中短暂显示
- 弹窗关闭时清除本地明文密码引用
- 密码字段默认掩码显示，点击才显示明文

**密码掩码显示规范**（Cryptographer 建议）：

```typescript
interface PasswordDisplayState {
  isMasked: boolean        // 默认 true
  showTimeout: number      // 明文显示超时（5秒）
  autoMaskOnBlur: boolean  // 失焦自动掩码（默认 true）
}
```

**行为定义**：
1. 编辑弹窗打开时，密码字段显示为 `****`（长度与实际密码一致）
2. 点击眼睛图标：切换为明文显示
3. 明文显示后：
   - 5秒无操作自动恢复掩码
   - 失去焦点立即恢复掩码
4. 弹窗关闭：清除所有明文引用

**安全流程**：
```
编辑弹窗打开:
  1. VaultStore.getDecryptedPassword(entryId)
  2. IPC 调用 crypto:decrypt，传入 encryptedPassword + masterKey
  3. 主进程解密返回 plaintext
  4. plaintext 存储在 VaultPage 的 editingDecryptedPassword ref
  5. AddPasswordModal 接收 decryptedPassword prop，默认掩码显示

编辑弹窗关闭:
  1. 清除 editingDecryptedPassword = null
  2. 清除 editingEntry = null
  3. 明文密码不再存在于渲染进程内存
```

### 剪贴板安全方案

**风险评估**：
| 风险 | 来源 | 等级 |
|------|------|------|
| 剪贴板历史记录 | Windows Clipboard History | 🟡 中 |
| 云同步剪贴板 | iCloud Clipboard | 🔴 高 |
| 第三方应用读取 | 剪贴板管理器 | 🟡 中 |

**安全措施**：

```typescript
// 复制后30秒自动清除剪贴板
setTimeout(async () => {
  try {
    await navigator.clipboard.writeText('')
    copiedEntryId.value = null
  } catch {
    copiedEntryId.value = null
  }
}, 30000)  // 30秒
```

**用户提示**：
- Toast消息："密码已复制，将在30秒后从剪贴板清除"
