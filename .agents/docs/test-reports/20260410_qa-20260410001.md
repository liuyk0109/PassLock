## 测试修复记录

### 修复1：Teleport/Transition 导致组件测试失败
**问题**：AddPasswordModal 和 ChangePasswordModal 使用 `<Teleport to="body">` 和 `<Transition>`，导致 `wrapper.find()` 无法找到 DOM 元素，且 DOM 内容渲染在 document.body 上造成跨测试污染。
**修复**：在 mount 配置中添加 stubs，将 Teleport 和 Transition 替换为 `<slot />` 模板，使内容渲染在 wrapper 内部。
```javascript
stubs: {
  Teleport: { template: '<slot />' },
  Transition: { template: '<slot />' },
}
```

### 修复2：Pinia 实例不一致
**问题**：测试中 `useVaultStore()` 使用的 Pinia 实例与 `mount()` 中 `plugins: [createPinia()]` 创建的实例不同，导致组件无法感知 store 状态变更（如 unlock）。
**修复**：在需要共享 store 状态的测试中，使用同一个 Pinia 实例：
```javascript
const pinia = createPinia()
setActivePinia(pinia)
const store = useVaultStore()
const wrapper = mount(Component, { global: { plugins: [pinia] } })
```

### 修复3：DOM 污染跨测试影响
**问题**：Teleport 内容在 `wrapper.unmount()` 后可能因 Transition 动画未完成而残留，影响后续测试。
**修复**：在 `afterEach` 中添加 `document.body.innerHTML = ''` 清理残留 DOM。

### 修复4：空表单提交测试逻辑错误
**问题**：AddPasswordModal 的保存按钮在表单无效时被 disabled，无法通过点击触发验证错误。
**修复**：改为验证按钮 disabled 状态。
