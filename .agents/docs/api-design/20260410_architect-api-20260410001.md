## 1. 新增IPC API

### 1.1 crypto:changeMasterPassword

**通道名**：`crypto:changeMasterPassword`

**方向**：渲染进程 → 主进程（invoke）

**描述**：执行主密码修改的完整流程，包括验证当前密码、备份数据、逐条重加密、更新验证数据、失败时自动回滚。

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| currentPassword | string | 是 | 当前主密码（明文） |
| newPassword | string | 是 | 新主密码（明文） |

#### 响应

```typescript
interface ChangeMasterPasswordResult {
  success: boolean
  error?: string  // 仅失败时返回
}
```

**成功响应**：
```json
{
  "success": true
}
```

**失败响应**：
```json
{
  "success": false,
  "error": "当前密码错误"
}
```

#### 错误码

| error值 | 说明 | 触发条件 |
|---------|------|----------|
| `"当前密码错误"` | 验证失败 | verifyPassword返回false |
| `"加密失败，数据已恢复"` | 重加密过程中异常 | decrypt/encrypt抛出异常 |
| `"保存失败，数据已恢复"` | 数据库写入失败 | setEntries/setMasterKeyVerify抛出异常 |
| `"未知错误"` | 未预料的异常 | 其他运行时错误 |

#### 主进程实现规范

```typescript
ipcMain.handle('crypto:changeMasterPassword',
  async (event, currentPassword: string, newPassword: string): Promise<ChangeMasterPasswordResult> => {
    let backup: { entries: VaultEntry[]; masterKeyVerify: string | null } | null = null

    try {
      // 1. 验证当前密码
      const verifyData = getMasterKeyVerify()
      if (!verifyData) {
        return { success: false, error: '验证数据不存在' }
      }

      const isValid = verifyPassword(verifyData, currentPassword)
      if (!isValid) {
        return { success: false, error: '当前密码错误' }
      }

      // 2. 备份当前数据（内存深拷贝）
      backup = {
        entries: JSON.parse(JSON.stringify(getEntries())),
        masterKeyVerify: getMasterKeyVerify()
      }

      // 3. 逐条重加密
      const entries = getEntries()
      for (let i = 0; i < entries.length; i++) {
        // 用旧密码解密
        const plaintext = decrypt(entries[i].password, currentPassword)
        // 用新密码加密
        entries[i].password = encrypt(plaintext, newPassword)

        // 发送进度事件
        event.sender.send('change-password-progress', i + 1, entries.length)
      }

      // 4. 批量写入数据库
      await setEntries(entries)

      // 5. 更新验证数据
      const newVerifyData = createVerifyData(newPassword)
      await setMasterKeyVerify(newVerifyData)

      return { success: true }

    } catch (error: any) {
      // 6. 失败时回滚
      if (backup) {
        try {
          await setEntries(backup.entries)
          await setMasterKeyVerify(backup.masterKeyVerify!)
        } catch (rollbackError) {
          console.error('Rollback failed:', rollbackError)
        }
      }

      const message = error instanceof Error ? error.message : '未知错误'
      return { success: false, error: message || '加密失败，数据已恢复' }

    } finally {
      // 7. 清除内存备份
      backup = null
    }
  }
)
```

## 2. 新增IPC事件

### 2.1 change-password-progress

**通道名**：`change-password-progress`

**方向**：主进程 → 渲染进程（send）

**描述**：主密码修改过程中的加密进度推送事件。

#### 事件数据

| 参数名 | 类型 | 说明 |
|--------|------|------|
| current | number | 当前已加密条目数（1-based） |
| total | number | 总条目数 |

**示例**：
```typescript
// 主进程发送
event.sender.send('change-password-progress', 3, 5)
// 表示：正在加密第3/5条目
```

#### Preload实现

```typescript
changeMasterPassword: (currentPassword: string, newPassword: string, onProgress?: (current: number, total: number) => void) => {
  const progressChannel = 'change-password-progress'

  // 注册进度监听
  if (onProgress) {
    const handler = (_event: any, current: number, total: number) => {
      onProgress(current, total)
    }
    ipcRenderer.on(progressChannel, handler)
  }

  // 调用主进程
  return ipcRenderer.invoke('crypto:changeMasterPassword', currentPassword, newPassword)
    .finally(() => {
      // 清理监听器
      ipcRenderer.removeAllListeners(progressChannel)
    })
}
```

```typescript
/**
 * 修改主密码
 * @param currentPassword 当前主密码
 * @param newPassword 新主密码
 * @param onProgress 进度回调（可选）
 * @throws Error 修改失败时抛出异常
 */
async function changeMasterPassword(
  currentPassword: string,
  newPassword: string,
  onProgress?: (current: number, total: number) => void
): Promise<void> {
  if (!masterKey.value) {
    throw new Error('Vault is locked')
  }

  const result = await window.electronAPI.crypto.changeMasterPassword(
    currentPassword,
    newPassword,
    onProgress
  )

  if (!result.success) {
    throw new Error(result.error || '修改密码失败')
  }

  // 更新内存中的主密码
  masterKey.value = newPassword

  // 重新加载条目（数据库中的条目已被新密码加密）
  await loadEntries()
}
```

#### setCurrentPage

```typescript
/**
 * 切换当前页面
 * @param page 目标页面
 */

  currentPage.value = page
}
```

## 8. 空密码库处理

当密码库中无条目（entries为空数组）时：

1. 验证当前密码 → 正常执行
2. 备份数据 → backup.entries = []
3. 重加密循环 → 跳过（0次迭代）
4. 不发送任何进度事件
5. 更新验证数据 → 正常执行
6. 返回成功

**特殊处理**：ChangePasswordModal中，当total=0时，进度条直接显示100%，文字显示"正在更新验证数据..."
