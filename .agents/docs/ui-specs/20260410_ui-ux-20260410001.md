# 主密码修改功能 UI设计规范

## 设计概述

本功能为PassLock密码管理器提供主密码修改功能，包含设置页面（Settings.vue）和修改密码弹窗（ChangePasswordModal.vue）两个核心组件。设计遵循现有项目的暗色主题风格，强调安全感与简约现代化。

---

## 布局结构

### 1. 设置页面（Settings.vue）

#### 页面导航
- **入口位置**：VaultHeader右侧按钮区域，在"锁定"按钮前添加"设置"按钮
- **路由方式**：App.vue中的页面切换（与VaultPage同级）
- **返回方式**：设置页面Header左侧提供"返回密码库"按钮

## 交互行为

### 设置页面交互

#### 1. 页面进入流程
- 用户点击VaultHeader"设置"按钮
- 页面使用Transition动画切换（与现有page动画一致）
- 设置页面首次加载时无额外数据请求

#### 2. 返回密码库
- 点击Header左侧"返回"按钮
- 使用Transition动画切换回VaultPage
- 保持解锁状态

#### 3. 修改密码入口点击
- 点击"修改主密码"行
- 弹出ChangePasswordModal弹窗
- 使用modal动画进入

### 修改密码弹窗交互

#### 1. 表单验证流程
```
用户输入当前密码 → 验证正确性（本地）
    ↓
输入新密码 → 实时显示强度指示器
    ↓
强度<medium → 显示警告提示，禁用确认按钮
    ↓
输入确认密码 → 实时匹配检查
    ↓
密码不匹配 → 显示错误提示，禁用确认按钮
    ↓
所有验证通过 → 启用确认按钮
```

#### 2. 密码可见性切换
- 点击眼睛图标切换显示/隐藏
- 默认隐藏状态
- 新密码输入框：点击后显示明文
- 确认密码输入框：点击后显示明文

#### 3. 提交流程
```
点击确认修改 → 显示进度覆盖层
    ↓
验证当前密码（后端）→ 失败时显示错误并恢复
    ↓
创建数据备份（内存）→ 失败时显示错误
    ↓
逐条重新加密 → 显示进度 "正在加密 X/Y"
    ↓
更新验证数据 → 失败时自动恢复备份
    ↓
成功 → 显示成功提示，关闭弹窗，更新状态
```

#### 4. 错误处理
- 当前密码验证失败：显示错误提示 "当前密码错误"
- 加密过程失败：显示错误提示 "加密失败，数据已恢复"
- 任何失败：自动恢复备份数据，用户可重新尝试

#### 5. 进度覆盖层
- 点击确认后立即覆盖整个弹窗内容
- 禁止关闭弹窗（ESC无效，点击背景无效）
- 显示加密进度条和当前进度文字
- 完成后自动关闭弹窗

#### 6. 关闭弹窗
- 取消按钮：直接关闭，无确认提示
- ESC键：关闭弹窗
- 点击背景：关闭弹窗
- 进度进行中：禁止关闭

## 状态定义

### 弹窗状态流转

| 状态 | 描述 | UI表现 |
|------|------|--------|
| initial | 初始打开 | 所有输入框为空，确认按钮禁用 |
| validating | 表单填写中 | 实时显示强度/匹配状态 |
| ready | 验证通过 | 确认按钮可用，可提交 |
| submitting | 提交中 | 显示进度覆盖层，禁止关闭 |
| success | 成功 | 关闭弹窗，显示成功toast |
| error | 失败 | 显示错误提示，恢复表单 |

### 输入框状态

| 状态 | 样式表现 |
|------|----------|
| empty | placeholder显示 |
| filled | 正常显示 |
| focus | border高亮，box-shadow |
| error | border变红，显示错误文字 |
| disabled | opacity: 0.6，cursor禁用 |

### 密码强度阈值

| 级别 | 分值范围 | 颜色 | 是否允许提交 |
|------|----------|------|--------------|
| 太短/弱 | 0-25 | #ef4444 | ❌ 禁止 |
| 中等 | 40-50 | #f59e0b | ✅ 允许 |
| 强 | 75 | #22c55e | ✅ 允许 |
| 非常强 | 100 | #14b8a6 | ✅ 允许 |

**关键规则**：强度必须 >= 40分（medium级别）才允许提交
