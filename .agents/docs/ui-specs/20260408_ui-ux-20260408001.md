## 一、需求分析

### 功能概述
用户空闲超过配置时间后自动锁定密码库，属于**后台静默行为**，无需复杂UI设计。

### 设计约束
| 约束项 | 约束内容 |
|--------|----------|
| 交互方式 | 静默锁定，无需倒计时提醒 |
| 设置界面 | 本次版本不实现，使用默认5分钟 |
| 现有组件 | LockScreen已完善，无需改动 |

### 用户场景
```
用户解锁密码库 → 使用应用 → 离开（空闲） → 5分钟后自动锁定 → 用户返回看到LockScreen
```

---

## 二、UI设计方案

### 2.1 核心结论

**本次任务无需新增UI组件**。原因：

1. **静默锁定需求**：用户明确不需要倒计时提醒，锁定过程无声无息
2. **现有过渡动画**：App.vue已实现平滑的页面切换动画（scale + opacity）
3. **LockScreen已完善**：现有锁定界面视觉设计完整，无需修改

### 2.2 过渡动画利用

自动锁定触发时，App.vue通过 `v-if="vaultStore.isLocked"` 切换组件，已有的过渡动画将平滑展示：

```css
/* 现有动画效果 */
.page-enter-active {
  animation: page-in 0.3s ease;
  /* opacity 0→1, scale 0.98→1 */
}
```

**用户体验**：用户返回时看到锁屏界面自然淡入，符合直觉。

### 2.3 可选优化项（建议跳过）

| 优化项 | 说明 | 建议 |
|--------|------|------|
| 锁定原因提示 | LockScreen显示"因空闲已自动锁定" | 暂不实现，保持简洁 |
| 锁定时间显示 | 显示"已锁定5分钟" | 不必要，增加复杂度 |
| 状态指示器 | 穗果栏小图标显示"已解锁"状态 | 后续版本考虑 |

## 六、交互行为说明

### 6.1 用户活动检测（后台）
```
鼠标移动 → 重置计时器
键盘输入 → 重置计时器
鼠标点击 → 重置计时器
```

### 6.2 锁定触发流程
```
空闲达到阈值 → vaultStore.lock() → isLocked=true → App.vue切换显示LockScreen → 过渡动画展示
```

### 6.3 解锁后重置
```
用户输入密码解锁 → vaultStore.unlock() → isLocked=false → 计时器从0开始重新检测
```
