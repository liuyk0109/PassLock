## 一、整体设计理念

### 1.1 品牌定位
- **产品类型**：安全密码管理器
- **核心价值**：安全感 + 简洁现代
- **视觉关键词**：Trustworthy、Minimal、Secure、Modern

### 1.2 设计原则
- **零新增组件原则**：最大化利用现有组件，避免增加维护负担
- **一致性优先**：所有改动需与现有Teal色系和暗色主题协调
- **渐进增强**：优先实现核心功能，可选效果在条件允许时实现

## 三、窗口外观优化

### 3.1 窗口配置规格

| 属性 | 值 | 实现方式 |
|------|-----|----------|
| 窗口圆角 | `16px` | Electron `roundedCorners` 或 CSS模拟 |
| 外部阴影 | `0 12px 40px rgba(0,0,0,0.4)` | CSS `box-shadow` 或系统API |
| 背景模糊 | `blur(16px)` | CSS `backdrop-filter` |

### 3.2 Electron窗口配置建议

```typescript
// electron/main.ts 建议配置
const win = new BrowserWindow({
  width: 800,
  height: 600,
  frame: false,           // 无边框（需自定义标题栏）
  transparent: true,      // 透明背景（支持圆角）
  vibrancy: 'under-window', // macOS 模糊效果
  visualEffectState: 'active',
  webPreferences: {
    preload: path.join(__dirname, 'preload.js'),
    contextIsolation: true,
    nodeIntegration: false,
  },
})

// 移除默认菜单栏
win.setMenu(null)
```

### 3.3 CSS模拟方案（降级）

当系统不支持原生圆角时，在`App.vue`添加：

```css
/* 窗口外观模拟 */
#app {
  border-radius: 16px;
  overflow: hidden;
  box-shadow: 
    0 0 0 1px rgba(255,255,255,0.06),
    0 12px 40px rgba(0,0,0,0.4);
}

/* 内容区域需配合圆角 */
.lock-screen,
.vault-page {
  border-radius: 16px;
}
```

### 3.4 跨平台兼容策略

| 平台 | 优先方案 | 降级方案 |
|------|----------|----------|
| macOS 11+ | 原生圆角 + vibrancy | CSS模拟 |
| Windows 11 | DWM圆角API | CSS模拟 |
| Windows 10 | CSS模拟 | 无圆角 |
| Linux(GNOME) | CSS模拟 | 无圆角 |

## 四、首屏错误 Toast 化

### 4.1 设计目标

- **位置**：卡片顶部边缘或屏幕顶部中央
- **行为**：3秒自动消失 + 可手动点击关闭
- **布局影响**：不撑开表单高度，使用fixed定位

### 4.2 Toast组件规格

```typescript
// Toast状态管理
interface ToastState {
  visible: boolean
  message: string
  type: 'error' | 'success' | 'warning'
  duration: number  // 默认3000ms
}
```

### 4.3 视觉规范

| 属性 | 值 | 说明 |
|------|-----|------|
| 定位 | `fixed` | 不影响布局 |
| 位置 | `top: 20px, left: 50%, transform: translateX(-50%)` | 屏幕顶部中央 |
| 宽度 | `max-width: 320px` | 适中宽度 |
| 圆角 | `10px` | 与error-box一致 |
| 背景 | `rgba(239, 68, 68, 0.15)` | 半透明红色 |
| 边框 | `1px solid rgba(239, 68, 68, 0.3)` | 红色边框 |
| 文字 | `#ef4444` | 错误色 |
| 阴影 | `0 4px 20px rgba(239,68,68,0.2)` | 红色光晕 |

### 4.4 动画规范

```css
/* Toast入场动画 */
.toast-enter-active {
  animation: toast-in 0.3s ease;
}

.toast-leave-active {
  animation: toast-out 0.25s ease forwards;
}

@keyframes toast-in {
  from {
    opacity: 0;
    transform: translateX(-50%) translateY(-20px);
  }
  to {
    opacity: 1;
    transform: translateX(-50%) translateY(0);
  }
}

@keyframes toast-out {
  from {
    opacity: 1;
    transform: translateX(-50%) translateY(0);
  }
  to {
    opacity: 0;
    transform: translateX(-50%) translateY(-20px);
  }
}
```

### 4.5 实现方案

**方案A（推荐）**：新建`Toast.vue`组件，使用`Teleport`到body
- 优点：复用性强，可全局调用
- 缺点：增加新组件

**方案B**：在LockScreen内部实现局部Toast
- 优点：零新增组件
- 缺点：仅限于LockScreen使用

**推荐方案A**，因为Toast是通用组件，后续其他页面也可使用。

## 五、密码库响应式布局优化

### 5.1 当前问题分析

**现有实现**（VaultPage.vue 402-486行）：
```css
/* 固定断点切换，跳变明显 */
.cards-grid {
  display: grid;
  grid-template-columns: repeat(var(--card-columns, 4), 260px);
}

@media (max-width: 1199px) { --card-columns: 3; }
@media (max-width: 1023px) { --card-columns: 2; }
@media (max-width: 639px)  { --card-columns: 1; }
```

### 5.2 优化方案：CSS Grid auto-fill

```css
/* 流畅自适应布局 */
.cards-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));
  gap: 24px 20px;
  justify-content: center;
  max-width: 1200px;
}

/* 卡片宽度自适应 */
.password-card {
  width: 100%;
  min-width: 240px;
  max-width: 320px;
}
```

### 5.3 视觉对比

| 方案 | 断点切换 | 流畅度 | 实现复杂度 |
|------|----------|--------|-----------|
| 现有固定断点 | 4→3→2→1 跳变 | 差 | 低 |
| auto-fill方案 | 流畅渐进 | 好 | 低 |

### 5.4 卡片最小宽度理由

- **240px**：保证标题（约10个汉字）完整显示
- **内容布局**：图标24px + 标题约150px + margin
- **触控友好**：移动端点击区域足够大

## 八、设计决策记录

### 决策1：图标风格选择
- **选择**：简约几何风格（锁形）
- **理由**：符合安全产品定位，与Teal色系协调，现代感强
- **对比**：复杂渐变风格会显得不专业，与暗色主题冲突

### 决策2：Toast消失策略
- **选择**：自动消失3秒 + 可手动关闭
- **理由**：平衡信息展示时长与用户操作自由度
- **对比**：永久停留会干扰后续操作，立即消失可能来不及阅读

### 决策3：响应式布局方案
- **选择**：CSS Grid auto-fill
- **理由**：原生支持流畅过渡，无需媒体查询跳变
- **对比**：固定断点方案跳变明显，用户体验差

### 决策4：窗口外观实现优先级
- **选择**：CSS模拟作为主要方案，原生API作为增强
- **理由**：跨平台兼容性优先，CSS方案可控性高
- **对比**：完全依赖原生API在Windows/Linux表现不一致
