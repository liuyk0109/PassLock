## 一、现有设计审查报告

### 1.1 交互逻辑问题

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 🔴 密码可见性切换缺失 | 高 | 用户无法确认输入是否正确，易输错导致解锁失败 |

| 问题 | 严重程度 | 说明 |

### 2.2 颜色系统

> **设计原则**: 避免蓝紫渐变等AI惯用色系，采用青绿色(Teal)为主色传达专业安全感，琥珀色(Amber)为辅色增加温暖感。

#### 主色系（青绿色 Teal - 安全感主题）
```css
/* 主色 - 青绿色传达专业安全感，区别于蓝紫AI色系 */
--primary-300: #5eead4;  /* 高亮状态 */
--primary-400: #2dd4bf;  /* 悬停高亮 */
--primary-500: #14b8a6;  /* 主按钮 */
--primary-600: #0d9488;  /* 悬停状态 */
--primary-700: #0f766e;  /* 激活状态 */
--primary-800: #047857;  /* 深色变体 */

/* Logo渐变 - 青绿渐变，无蓝紫成分 */
--primary-gradient: linear-gradient(135deg, #14b8a6 0%, #0d9488 100%);
```

/* 辅色 - 琥珀色增加温暖感，避免冷冰冰科技感 */

--accent-600: #d97706;  /* 深色 */

### 3.4 密码强度指示器

**位置**: 主密码输入框下方，间距4px

**结构**: 进度条 + 文字标签

**进度条**:
- 高度: 4px
- 背景轨道: `rgba(255,255,255,0.1)`
- 填充: 根据强度等级变化颜色
- 圆角: 2px

**强度等级计算**:
| 条件 | 等级 | 文字 | 颜色 |
|------|------|------|------|
| 长度≥6 + (大小写混合/数字/符号任意1项) | 弱(1) | "弱" | #ef4444 |
| 长度≥6 + 2项混合 | 中等(2) | "中等" | #f59e0b |
| 长度≥10 + 3项混合 | 强(3) | "强" | #22c55e |
| 长度≥10 + 全部混合 | 非常强(4) | "非常强" | #14b8a6 |

**混合条件检测项**:
1. 大小写混合: `/[a-z]/.test(pwd) && /[A-Z]/.test(pwd)`
2. 包含数字: `/[0-9]/.test(pwd)`
3. 包含符号: `/[^a-zA-Z0-9]/.test(pwd)`

## 四、交互行为

### 4.1 页面加载流程

1. 显示加载spinner + "正在初始化..."
2. 调用 `getMasterKeyVerify()` 检查密码库状态
3. 根据结果切换到 "首次创建" 或 "解锁" 表单

### 4.2 首次创建流程

```
用户输入主密码
  → 实时显示密码强度指示器
  → 用户点击眼睛图标可查看密码
  
用户输入确认密码
  → 实时显示匹配状态提示
  → 用户点击眼睛图标可查看密码

点击"创建密码库"
  → 按钮显示加载状态
  → 验证密码长度≥6
  → 验证两次密码匹配
  → 创建验证数据并存储
  → 解锁密码库
  
出错时显示错误提示框
```

### 4.3 解锁流程

```
用户输入主密码
  → 用户点击眼睛图标可查看密码
  
点击"解锁"
  → 按钮显示加载状态
  → 验证密码不为空
  → 获取验证数据
  → 验证密码正确性
  → 解锁密码库
  
出错时显示错误提示框
```

### 4.4 状态变化说明

| 用户操作 | 状态变化 |
|----------|----------|
| 输入主密码 | 密码强度指示器实时更新 |
| 输入确认密码 | 匹配状态实时更新 |
| 点击眼睛图标 | 切换密码可见性 |
| 点击按钮 | 按钮进入加载状态 |
| 输入时存在错误 | 清除错误提示 |
| 提交失败 | 显示错误提示框 |

## 七、设计决策记录

| 决策 | 理由 |
|------|------|
| 选择青绿色主色 | 避免蓝紫渐变等AI惯用色系；青绿色传达专业安全感，同时保持独特性 |
| 选择琥珀色辅色 | 增加温暖感，避免冷冰冰的科技感，让用户感到亲切 |
| 添加密码强度指示器 | 帮助用户设置安全的主密码，降低后续解锁失败风险 |
| 添加密码可见性切换 | 减少用户输错概率，提升首次设置成功率 |
| 使用毛玻璃卡片 | 保持现代感，同时与深色背景形成层次对比 |
| 底部安全提示 | 强化产品安全感定位，建立用户信任 |
