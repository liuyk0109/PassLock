## 功能需求

### 1. 产品图标设计
- **描述**：设计专业的品牌图标，用于打包成品、主页面、任务栏等位置展示
- **风格**：简约几何风格，使用简洁的几何形状和锁具元素，现代感强
- **应用位置**：
  - 打包产物图标（Windows .exe、macOS .dmg、Linux .AppImage）
  - 应用任务栏图标
  - 主页面 Logo 区域
- **输出形式**：
  - SVG 源文件（用于 Vue 组件）
  - PNG 多尺寸（用于打包：16x16、32x32、48x48、64x64、128x128、256x256、512x512）
  - AI 图像生成 Prompt（备选方案）

### 2. 移除默认菜单栏 + 窗口外观优化
- **描述**：移除 Electron 默认的 File/Edit/View/Windows/Help 菜单栏，为产品添加现代化窗口外观
- **验收标准**：
  - 无默认菜单栏显示
  - 窗口边缘圆角 16px
  - 窗口外部阴影效果（深度约 8-12px，柔和扩散）
  - 跨平台兼容（Windows/macOS/Linux）

### 3. 首屏错误展示优化
- **描述**：将首屏（LockScreen）的静态错误框改为短暂浮窗 Toast，避免撑开页面布局
- **当前问题**：`error-box` 会增加表单高度，影响视觉稳定性
- **验收标准**：
  - 错误提示以 Toast 浮窗形式展示
  - 浮窗位于卡片顶部或屏幕顶部
  - 自动消失时间：3秒
  - 不影响页面布局稳定性
  - 支持多个错误顺序展示（如有）

### 4. 密码库响应式布局优化
- **描述**：优化密码库页面的卡片网格布局，使窗口缩放时行列调整更流畅合理
- **当前问题**：使用固定断点切换列数（4→3→2→1），切换时跳变明显
- **验收标准**：
  - 使用 CSS Grid `auto-fill` 或类似技术实现流畅自适应
  - 卡片标签允许在一定范围内调整大小
  - 卡片最小宽度不低于 240px（保证内容可读）
  - 缩放过程流畅无跳变

### 5. 新增按钮图标重设计
- **描述**：重新设计密码库页面的「新增」按钮图标
- **当前问题**：使用多笔画加号图标，视觉不够简洁现代
- **验收标准**：
  - 新图标简洁现代，符合整体设计语言
  - 与锁定、设置按钮图标风格统一
  - 清晰传达「添加」语义

### 6. 其他 UI/UX 优化建议（由 UI/UX Agent 补充）
- 基于整体设计审查提出的改进建议
- 潜在方向：
  - Loading 状态优化（骨架屏 vs 当前 spinner）
  - 空状态图标优化
  - 按钮交互反馈增强
  - 颜色对比度优化（可访问性）
  - 弹窗动画优化
  - 搜索栏交互优化

## 非功能需求

### 性能要求
- Toast 动画流畅，FPS ≥ 60
- 响应式布局重排无明显延迟
- 图标资源加载不影响首屏时间

### 安全要求
- 不涉及安全功能修改
- 保持现有加密逻辑不变

### 兼容性要求
- Windows 10/11：圆角和阴影效果正常
- macOS 11+：圆角和阴影效果正常
- Linux（GNOME/KDE）：尽量兼容，可接受降级效果

---

## 约束条件

### 技术约束
- Electron 窗口外观受系统限制，部分效果可能需要 CSS 模拟
- 无边框窗口可能影响窗口拖拽和系统按钮，需谨慎处理
- 图标生成：若无法直接生成图片，提供 AI Prompt 由用户生成

### 时间约束
- 本任务为 v0.1.0-beta 最后一项，完成后进入打包发布阶段

### 资源约束
- 无专业设计软件资源，需依靠代码实现或 AI 辅助

---

## 验收标准

1. 产品图标已应用到打包产物和主页面
2. 默认菜单栏已移除，窗口外观符合设计（圆角16px + 强阴影）
3. 首屏错误以 Toast 形式展示，3秒自动消失，不影响布局
4. 密码库页面缩放流畅，卡片自适应调整
5. 新增按钮图标简洁现代
6. 其他 UI/UX 问题已识别并提供解决方案
