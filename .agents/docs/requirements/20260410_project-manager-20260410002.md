# 数据备份与恢复功能

## 需求概述
- 需求ID：20260410002
- 创建时间：2026-04-10
- 需求来源：v0.1.0-beta Milestone（P0优先级）
- 关联任务：`.agents/tasks/task_20260410002.md`

## 功能需求

### 1. 数据导出功能
- **描述**：将密码库数据导出为加密JSON文件，用户可选择保存位置
- **验收标准**：
  - 导出内容包含全部密码条目和验证数据（masterKeyVerify）
  - 弹出系统文件保存对话框，用户选择保存位置和文件名
  - 导出的JSON文件中密码条目保持加密状态
  - 验证数据一并导出，确保跨设备恢复可用
  - 导出成功后显示成功提示
  - 默认文件名格式：`PassLock_Backup_YYYYMMDD_HHMMSS.json`

### 2. 数据导入功能
- **描述**：从备份JSON文件导入数据恢复密码库
- **验收标准**：
  - 弹出系统文件选择对话框，用户选择备份文件
  - 导入前验证主密码是否与备份文件匹配
  - 主密码验证失败时显示错误提示，拒绝导入
  - 导入时检测条目ID冲突
  - 存在冲突时弹出冲突处理对话框，逐条询问用户覆盖或跳过
  - 无冲突的条目自动导入
  - 导入成功后显示导入条目数量统计

### 3. 导入冲突处理弹窗
- **描述**：导入时遇到同ID条目，弹出对话框让用户逐条选择处理方式
- **验收标准**：
  - 显示冲突条目列表（标题、用户名、创建时间）
  - 每条冲突条目提供"覆盖"、"跳过"、"保留两者"选项
  - "保留两者"时为新条目分配新ID
  - 显示冲突条目数量统计
  - 提供"全部覆盖"、"全部跳过"快捷按钮
  - 用户确认后执行导入操作

### 4. 设置页面集成
- **描述**：在设置页面添加导出/导入功能入口
- **验收标准**：
  - 设置页面显示"导出数据"按钮
  - 设置页面显示"导入数据"按钮
  - 导入按钮仅在解锁状态下可用
  - 导出按钮仅在解锁状态下可用

## 非功能需求

### 性能要求
- 导出操作：1000条目内完成时间<3秒
- 导入操作：1000条目内完成时间<5秒
- 大文件导入时显示进度提示

### 安全要求
- 导出文件中密码保持加密状态，不暴露明文
- 导入前必须验证主密码
- 验证数据导出，确保备份文件可用同一主密码恢复
- 文件传输过程不产生临时文件残留

### 兼容性要求
- 导出JSON格式跨平台兼容（Windows/macOS/Linux）
- 导入时兼容旧版本导出格式
- 支持空密码库导出（仅导出验证数据）
- 支持导入空备份文件（仅恢复验证数据）

## 约束条件

### 技术约束
- 使用Electron的dialog API进行文件选择
- JSON格式便于跨设备迁移和手动查看
- 备份文件无需额外加密（密码条目已加密）

### 时间约束
- 属于v0.1.0-beta版本，优先级P0

### 资源约束
- 无额外依赖要求

## 验收标准

1. 用户可通过设置页面点击"导出数据"按钮导出备份
2. 导出时弹出文件保存对话框，用户可选择保存位置
3. 导出的JSON文件包含全部条目和验证数据
4. 导出文件中密码保持加密状态
5. 用户可通过设置页面点击"导入数据"按钮导入备份
6. 导入时弹出文件选择对话框
7. 导入前验证主密码，密码不匹配时拒绝导入
8. 导入时遇到冲突条目弹出冲突处理对话框
9. 冲突对话框支持逐条选择覆盖/跳过/保留两者
10. 导入成功后显示导入条目数量统计
11. 空密码库导出/导入功能正常工作

## 备注

### 现有基础设施
- 数据库：`database.ts` 已有 `getEntries()`、`setEntries()`、`getMasterKeyVerify()`、`setMasterKeyVerify()`

- Electron dialog API需在主进程调用，渲染进程通过IPC通信
- 大文件导入时内存占用需注意
- 冲突处理逻辑复杂度较高
