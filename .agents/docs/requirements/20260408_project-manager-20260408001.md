# 自动锁定功能需求

## 需求概述
- 需求ID：20260408001
- 创建时间：2026-04-08
- 需求来源：用户描述 / v0.1.0-beta Milestone规划

## 功能需求

### 功能点1：空闲检测机制
- 描述：在Vue应用中监听用户活动（鼠标移动、键盘输入、点击），检测用户空闲状态
- 验收标准：
  1. 监听鼠标移动、键盘输入、鼠标点击事件
  2. 用户有任何活动时重置空闲计时器
  3. 计时器从用户最后一次活动开始计算
  4. 应用窗口最小化时暂停检测（可选优化）

### 功能点2：自动锁定触发
- 描述：用户空闲超过配置的时间后，自动调用 `vaultStore.lock()` 锁定密码库
- 验收标准：
  1. 空闲时间达到 `settings.autoLockTimeout`（分钟）后触发锁定
  2. 锁定后清除剪贴板中的密码数据
  3. 锁定后清除内存中的主密钥和密码条目
  4. 锁定后显示 LockScreen 组件要求重新解锁

### 功能点3：锁定时间配置
- 描述：使用数据库中已有的 `settings.autoLockTimeout` 字段，默认值为5分钟
- 验收标准：
  1. 从数据库读取 `settings.autoLockTimeout` 配置值
  2. 支持以分钟为单位配置（默认5分钟）
  3. 本次版本使用默认值，后续版本提供设置界面自定义

### 功能点4：解锁后重置计时器
- 描述：用户成功解锁密码库后，重置空闲计时器开始新的检测周期
- 验收标准：
  1. 解锁成功后立即启动空闲检测计时器
  2. 计时器从0开始重新计算空闲时间

## 非功能需求
- 性能要求：空闲检测不影响应用性能，事件监听采用节流策略
- 安全要求：锁定必须彻底清除所有敏感数据（主密钥、密码条目、剪贴板）
- 兼容性要求：在Windows/macOS/Linux上正常工作

## 约束条件
- 技术约束：使用前端渲染进程实现，不依赖Electron主进程powerMonitor
- 时间约束：v0.1.0-beta版本必须实现
- 资源约束：无

## 验收标准
1. 用户解锁密码库后，空闲计时器正常启动
2. 用户有任何活动（鼠标、键盘、点击）时，计时器重置
3. 空闲时间达到配置值后，密码库自动锁定
4. 锁定后所有敏感数据被清除，显示LockScreen
5. 设置界面暂不实现，使用默认5分钟锁定时间

## 备注
- 数据库已有 `settings.autoLockTimeout` 字段，默认值为5分钟
- `vaultStore.lock()` 方法已实现清除剪贴板和敏感数据的逻辑
- 后续版本可扩展：设置界面自定义锁定时间、系统级别空闲检测
