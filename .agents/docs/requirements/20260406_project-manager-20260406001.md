# 密码卡片操作按钮功能需求

## 需求概述
- 需求ID：20260406001
- 创建时间：2026-04-06
- 需求来源：用户描述

## 功能需求

### 功能点1：Hover显示操作按钮
- 描述：密码卡片上的操作按钮（删除、编辑、复制）默认隐藏，仅在鼠标悬停到卡片时才显示，保持界面清爽
- 验收标准：
  1. 鼠标未悬停时，操作按钮不可见
  2. 鼠标移入卡片后，操作按钮以平滑动画显示
  3. 鼠标移出卡片后，操作按钮以平滑动画隐藏
  4. 按钮排列顺序：编辑、复制、删除（从左到右）

### 功能点2：编辑密码条目
- 描述：点击编辑按钮后，打开编辑弹窗修改密码条目的名称、网站、密码、备注等信息
- 验收标准：
  1. 点击编辑按钮打开弹窗，弹窗标题显示"编辑密码条目"
  2. 弹窗预填充当前条目的所有信息（名称、网站、密码需解密显示、备注）
  3. 保存时重新加密密码并更新数据库
  4. 更新成功后卡片显示最新信息
  5. 可取消编辑关闭弹窗不做任何修改

### 功能点3：删除密码条目
- 描述：点击删除按钮后，弹出确认对话框，用户确认后删除该密码条目
- 验收标准：
  1. 点击删除按钮弹出确认对话框，显示"确定删除该密码条目吗？删除后无法恢复。"
  2. 确认对话框有"取消"和"删除"两个按钮
  3. 点击"取消"关闭对话框，不做任何操作
  4. 点击"删除"执行删除，从卡片网格中移除该卡片（带动画效果）
  5. 删除成功后从数据库中移除该条目

### 功能点4：复制密码功能保持
- 描述：保留现有的复制密码功能，改为hover时显示
- 验收标准：
  1. 复制按钮在hover时显示
  2. 点击复制按钮复制密码到剪贴板
  3. 复制成功显示"已复制"状态反馈

## 非功能需求
- 性能要求：操作按钮显示/隐藏动画流畅（≤200ms）
- 安全要求：编辑弹窗中显示密码需解密，关闭后清除
- 兼容性要求：响应式布局下按钮仍可正常操作

## 约束条件
- 技术约束：复用现有 AddPasswordModal 组件扩展编辑模式
- 时间约束：无
- 资源约束：无

## 验收标准
1. 密码卡片hover时正确显示三个操作按钮（编辑、复制、删除）
2. 编辑功能正确打开弹窗、预填充数据、保存更新
3. 删除功能正确弹出确认框、执行删除、移除卡片
4. 复制功能正常工作
5. 所有操作都有适当的视觉反馈和动画效果

## 备注
- 编辑弹窗复用 AddPasswordModal，通过 props 区分新增/编辑模式
- 删除确认弹窗可使用原生 confirm 或自定义确认组件
