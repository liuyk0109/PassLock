# 主密码修改功能

## 需求概述
- 需求ID：20260410001
- 创建时间：2026-04-10
- 需求来源：v0.1.0-beta Milestone（P0优先级）
- 关联任务：`.agents/tasks/task_20260410001.md`

## 功能需求

### 1. 主密码修改入口
- **描述**：在独立设置页面（Settings.vue）中提供主密码修改入口
- **验收标准**：
  - 用户解锁后可通过VaultHeader的设置按钮进入设置页面
  - 设置页面显示"修改主密码"按钮
  - 点击按钮弹出修改密码弹窗

### 2. 密码修改弹窗
- **描述**：弹窗包含当前密码验证、新密码输入、确认密码输入
- **验收标准**：
  - 显示当前密码输入框（用于验证）
  - 显示新密码输入框（带密码强度指示器）
  - 显示确认密码输入框（需与新密码一致）
  - 新密码强度低于medium时禁止提交，显示错误提示
  - 提供取消和确认按钮

### 3. 重新加密流程
- **描述**：用新主密码重新加密所有密码条目
- **验收标准**：
  - 验证当前密码正确后开始加密
  - 显示进度条："正在加密 X/Y 条目"
  - 逐条解密旧密码 → 用新密码加密
  - 更新数据库中的所有条目
  - 更新masterKeyVerify验证数据

### 4. 自动备份回滚
- **描述**：修改前自动备份原数据，失败时自动恢复
- **验收标准**：
  - 开始修改前自动创建数据备份（entries + masterKeyVerify）
  - 备份存储在临时位置（如内存或临时文件）
  - 任何步骤失败时自动恢复备份数据
  - 成功完成后清除备份

### 5. 状态更新
- **描述**：修改成功后更新内存中的主密码状态
- **验收标准**：
  - 成功后更新vaultStore.masterKey为新密码
  - 保持解锁状态不锁定密码库
  - 显示成功提示消息

## 非功能需求

### 性能要求
- 加密进度响应时间：每条目加密<100ms
- UI进度更新频率：每加密1条目更新一次

### 安全要求
- 新密码强度必须>=40分（medium级别）
- 失败时必须完全恢复原数据，不留残留
- 内存中的备份数据完成后立即清除

### 兼容性要求
- 兼容现有加密模块（AES-256-GCM）
- 兼容现有数据库结构（无需修改schema）
- 支持空密码库（无条目时仅更新验证数据）

## 约束条件

### 技术约束
- 使用现有加密API（crypto.encrypt/decrypt）
- 使用现有数据库API（db.getEntries/setEntries）
- 需新增IPC API用于备份/恢复操作

### 时间约束
- 属于v0.1.0-beta版本，优先级P0

### 资源约束
- 无额外依赖要求

## 验收标准

1. 用户可通过设置页面进入主密码修改功能
2. 输入当前密码可正确验证
3. 新密码强度低于medium时拒绝修改
4. 修改过程显示加密进度条
5. 修改成功后所有条目可用新密码正常解密
6. 修改失败时数据完整恢复，无丢失
7. 新密码强度指示器正确显示强度等级
8. 条目数量为0时修改功能正常工作（仅更新验证数据）

## 备注

### 现有基础设施
- 加密模块：`crypto.ts` 已有 encrypt/decrypt/verifyPassword/createVerifyData
- 数据库：`database.ts` 已有 masterKeyVerify字段、entries数组
- IPC API：`preload.ts` 已暴露 crypto/db 相关方法

### 技术风险
- 大量条目时加密耗时可能较长，需进度反馈
- 内存中暂存解密数据需注意安全处理
- 失败恢复需确保数据完整性
