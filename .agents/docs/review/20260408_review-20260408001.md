## ⚠️ 警告（应该修复）

### 1. 配置加载与解锁的竞态条件

**位置**：[src/App.vue#L46-L48](/d:/code/PassLock/src/App.vue)

**问题描述**：
`initialize()` 是异步方法，从数据库加载 `autoLockTimeout` 配置。但用户可能在配置加载完成前就解锁密码库，导致空闲检测使用默认值 5 分钟，而非用户实际配置的值。

**执行顺序风险**：
1. `App.vue` 的 `onMounted` 开始执行 `await vaultStore.initialize()`
2. 同时 `LockScreen.vue` 的 `onMounted` 也开始执行 `getMasterKeyVerify()`
3. 如果 `getMasterKeyVerify()` 比 `initialize()` 更快完成，用户可以立即解锁
4. `unlock()` 调用 `startIdleDetection()` 时，`autoLockTimeout` 仍是默认值 5

**修复方案**：

```typescript
// vault.ts - 添加初始化标记
const isInitialized = ref(false)

async function initialize(): Promise<void> {
  await loadAutoLockTimeout()
  isInitialized.value = true
}

async function unlock(key: string) {
  // 等待初始化完成（防止竞态条件）
  if (!isInitialized.value) {
    await initialize()
  }
  masterKey.value = key
  isLocked.value = false
  startIdleDetection()
}
```

**影响**：中等 - 可能导致空闲锁定时间与用户配置不符

## 💡 建议（可选改进）

### 1. throttle 函数类型定义

**位置**：[src/App.vue#L10-L18](/d:/code/PassLock/src/App.vue)

**问题**：使用 `Function` 类型过于宽松，TypeScript 中建议使用更具体的函数类型签名。

**改进方案**：

```typescript
function throttle<T extends (...args: unknown[]) => void>(fn: T, delay: number): T {
  let lastCall = 0
  return ((...args: unknown[]) => {
    const now = Date.now()
    if (now - lastCall >= delay) {
      lastCall = now
      fn(...args)
    }
  }) as T
}
```
