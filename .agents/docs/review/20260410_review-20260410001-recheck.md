## 💡 建议（可选优化）

### `backup.masterKeyVerify!` 非空断言可优化为类型安全写法

**位置**：[electron/main.ts#L115](/d:/code/PassLock/electron/main.ts)

`backup.masterKeyVerify` 类型为 `string | null`，虽然逻辑上 `!` 断言安全（第72-74行已验证 `verifyData` 非空），但可复用已验证变量消除非空断言：

```typescript
// 当前写法
backup = {
  entries: JSON.parse(JSON.stringify(getEntries())),
  masterKeyVerify: getMasterKeyVerify()  // 返回 string | null
}

// 建议写法：复用已验证的 verifyData 变量
backup = {
  entries: JSON.parse(JSON.stringify(getEntries())),
  masterKeyVerify: verifyData  // 已确认为 string
}
```

同时可将 `backup` 的类型从 `{ entries: VaultEntry[]; masterKeyVerify: string | null }` 收窄为 `{ entries: VaultEntry[]; masterKeyVerify: string }`，消除回滚路径中的 `!` 断言。

**优先级**：低 — 不影响功能和安全，仅类型优雅性改进
