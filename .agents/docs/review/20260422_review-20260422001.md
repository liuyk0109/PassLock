## 警告级别（SHOULD FIX）

### 1. Toast 定时器双重管理导致竞态条件

**文件**：[vault.ts](/d:/code/PassLock/src/stores/vault.ts#L451-L457)

**问题**：`showToast` 函数和 `Toast.vue` 组件各自独立设置了 3秒自动隐藏的 `setTimeout`，导致双重定时器竞态。

vault.ts 中的 `showToast` 函数：
```typescript
function showToast(type: 'success' | 'error', title: string, details?: string) {
  toast.value = { visible: true, type, title, details }
  // 3秒后自动隐藏 ← 问题：这里设置了一个定时器
  setTimeout(() => {
    toast.value.visible = false
  }, 3000)
}
```

同时 Toast.vue 组件的 watch 也设置了定时器：
```typescript
watch(() => vaultStore.toast.visible, (visible) => {
  if (visible) {
    // ...
    hideTimer = window.setTimeout(() => {
      vaultStore.toast.visible = false
    }, 3000)  // ← 这里又设置了一个定时器
  }
})
```

**影响**：
- 两个定时器同时尝试修改 `toast.visible`，可能导致状态不一致
- 用户手动关闭 Toast 后，vault.ts 的定时器仍会执行（虽然无害但浪费资源）
- 违反单一职责原则，定时器管理应该集中在一个地方

**修复建议**：移除 vault.ts 中 `showToast` 函数内的 setTimeout，让 Toast.vue 组件完全负责定时器逻辑：

```typescript
// vault.ts - showToast 函数修改为：
function showToast(type: 'success' | 'error', title: string, details?: string) {
  toast.value = { visible: true, type, title, details }
  // 移除 setTimeout，由 Toast.vue 组件负责自动隐藏
}
```
