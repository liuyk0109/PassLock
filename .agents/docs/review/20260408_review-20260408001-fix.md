### 2. throttle 泛型类型修复 ✓ 已解决

**App.vue 修复实现：**
```typescript
function throttle<T extends (...args: unknown[]) => void>(fn: T, delay: number): T {
  let lastCall = 0
  return ((...args: unknown[]) => {
    const now = Date.now()
    if (now - lastCall >= delay) {
      lastCall = now
      fn(...args)
    }
  }) as T
}
```

**验证结论**：类型定义已优化，泛型 `T` 保留原函数类型签名，类型安全性提升。

## 💡 优化建议（可选）

### 避免 initialize() 并发重复调用

**位置**：[vault.ts#L136-L139](/d:/code/PassLock/src/stores/vault.ts)

**背景**：如果 App.vue 的 `onMounted` 正在执行 `initialize()` 时用户同时解锁，`unlock()` 会再次调用 `initialize()`，导致 `loadAutoLockTimeout()` 被并发执行两次。

**可选方案**：
```typescript
let initPromise: Promise<void> | null = null

async function initialize(): Promise<void> {
  if (isInitialized.value) return
  if (initPromise) return initPromise
  
  initPromise = loadAutoLockTimeout().then(() => {
    isInitialized.value = true
    initPromise = null
  })
  return initPromise
}
```

**注意**：这是优化建议，不是必须修复的问题。当前实现已能满足功能需求，`loadAutoLockTimeout()` 是幂等的。
