### 1. 导入操作非原子性——逐条写入数据库

**修复状态**：✅ **已修复**

**修复代码位置**：`electron/main.ts#L397-L426`

**修复方案**：
```typescript
// 7. 在内存中完成所有条目修改（原子性）
const database = getDatabase()
const now = Date.now()

for (const entry of backupData.entries) {
  if (localIdSet.has(entry.id)) {
    // 冲突条目处理（内存修改）
    const action = resolutionMap.get(entry.id) ?? 'skip'
    if (action === 'override') {
      const index = database.data.entries.findIndex(e => e.id === entry.id)
      if (index !== -1) {
        database.data.entries[index] = { ...entry, updatedAt: now }
        overridden++
      }
    } else if (action === 'keepboth') {
      database.data.entries.push({ ...entry, id: crypto.randomUUID() })
      keptBoth++
    } else {
      skipped++
    }
  } else {
    // 无冲突条目 - 直接添加到内存
    database.data.entries.push(entry)
    added++
  }
}

// 8. 单次原子写入
await database.write()
```

**验证结论**：
- ✅ 不再逐条调用 `addEntry`/`updateEntry`（这些会触发多次数据库写入）
- ✅ 所有修改在内存中完成后执行单次 `database.write()`
- ✅ 保留回滚机制和内存清理
- ✅ 参照 `changeMasterPassword` 的原子性模式实现
