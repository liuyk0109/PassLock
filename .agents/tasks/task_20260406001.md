## 关键决策记录

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 编辑弹窗方案 | 复用 AddPasswordModal | 代码精简，避免重复，通过 props 区分模式 |
| 删除确认方案 | 二次确认弹窗 | 防止误删，提高安全性 |
| 操作按钮顺序 | 编辑、复制、删除 | 安全操作在前，危险操作在后 |
| IPC API 方案 | 复用现有设计 | crypto.encrypt/decrypt 和 db.updateEntry/deleteEntry 已完备 |
| 状态管理方案 | VaultPage 本地管理 | 编辑/删除确认状态是 UI 层逻辑，不应放入全局 Store |
| 密码解密流程 | VaultStore.getDecryptedPassword | 解密操作在主进程执行，渲染进程通过 IPC 获取明文 |
| IV长度修正 | 16字节→12字节 | NIST SP 800-38D推荐值，AES-GCM最佳实践 |
| 编辑密码显示 | 默认掩码+点击显示+自动隐藏 | 防止屏幕录制/截图捕获 |
| 剪贴板安全 | 复制后30秒自动清除 | 平衡安全性与用户体验 |
| 密钥传递声明 | 修正为如实说明 | 架构文档与实际实现保持一致，IPC在可信边界内 |
