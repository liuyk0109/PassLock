### 2026-04-10 - 代码审查完成
- 完成数据备份与恢复功能的代码审查
- 审查报告：`.agents/docs/review/20260410_review-20260410002.md`
- 审查结论：发现 2 个严重问题、3 个警告、1 个建议
- **严重问题**：
  1. 导入未验证备份主密码与当前密码库主密码一致，可导致导入条目永久不可解密
  2. `importData` 未验证备份文件格式，可处理恶意/损坏文件
- **警告**：
  1. 导入操作非原子性——逐条写入数据库，崩溃可导致数据不一致
  2. 冲突处理弹窗中"更改"按钮无效——无法撤销已选择的冲突解决方案
  3. `dialog:showSaveDialog/showOpenDialog` 参数类型为 `any`，存在安全风险
- **建议**：
  1. `ConflictCheckResult` 类型与架构文档不一致（缺少 `noConflictIds` 字段）
- **审查通过项**：安全分层设计、备份密码加密、主密码修改原子性、导入状态机完整性、版本兼容性检查、IPC命名空间拆分、测试覆盖（共7项）

## 关键决策记录

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 导出范围 | 全部数据 | 完整备份可跨设备恢复，不丢失验证数据 |
| 导出位置 | 用户选择位置 | 灵活方便，用户可自定义备份存储位置 |
| 冲突策略 | 用户逐条选择 | 精细控制，避免误覆盖或误跳过重要条目 |
| 文件格式 | 加密JSON | 安全性高，密码已加密，需主密码才能解密 |
| IPC粒度 | 7个细粒度通道 | 安全分层、流程可控、数据最小化 |
| 冲突检测位置 | 主进程 | 密码字段不传渲染进程，数据最小化 |
| 导入状态管理 | 集成vaultStore | 逻辑内聚，避免Store间通信 |
| 加密方案 | 复用现有crypto | 无需新增，verifyPassword已满足 |
